编译的问题

Posted on 2020-07-23 | In c | | 次
c的编译过程和常遇到的问题以及解决方法

一个C的项目(project)包含多个源文件(cpp)；

C项目的编译，是以源文件(cpp)为基本单位进行编译和连接的。

c 的编译过程

  • 预处理Preprocessing (gcc -E hello -o hello.i)

    (1)宏的展开(#define)

    (2)条件编译的处理(#if, #ifndef, #ifdef, #endif)

    (3)头文件的展开(#include)

  • 编译Compilation (gcc -S hello.i -o hello.s)

    (1)把预处理之后的.i文件编译成汇编文件(.S)

    (2)编译过程就是把预处理完的文件进行一系列词法分析，语法分析，语义分析及优化后生成相应的汇编文件

  • 汇编Assembly (gcc -c hello.s -o hello.o)

    (1)把编译后的汇编文件(.S)汇编成目标文件(.o)

    (2)汇编器是将汇编代码转变成机器可以执行的命令，每一个汇编语句几乎都对应一条机器指令

    (3)得到一个.o为后缀的目标文件，由于hello.o的内容为机器码，不能以文本形式方便的呈现。可用命令hexdump hello.o 打开

  • 连接Linking (gcc hello.o -o hello)

    把汇编之后的目标文件(.o)连接成可执行文件

编译时遇到的问题

在编译C项目的时候，是以源文件(cpp)为基本单位进行编译的

在预处理的时候，源文件(.cpp)中的头文件(.h)会嵌套展开

预处理的的命令格式：gcc -E hello -o hello.i

如果处理不当，可能造成一些编译错误，以下是一些常见的编译错误。

头文件重复包含问题：

问题出现的原因

情况一：源文件中的一个头文件，被源文件中的其他头文件包含

源文件(a.cpp)中包含了头文件(a.h和b.h)

头文件(a.h)中又包含了头文件(b.h)

预处理的时候，源文件(a.cpp)会重复包含头文件(b.h)中的test_b函数，导致编译错误。

注意：在头文件(.h)，如果是只是声明函数而不是定义函数，则不会出现头文件重复包含的问题

情况二：源文件中的多个头文件，共同包含相同的头文件

源文件(main.cpp)中包含了头文件(a.h)和头文件(b.h)

头文件(a.h)中包含了头文件(c.h)

头文件(b.h)中包含了头文件(c.h)

预处理的时候，源文件(main.cpp)会重复包含头文件(c.h)的test_c函数，导致编译错误。

解决方式：

在头文件中加入#pragam once或者使用条件编译语句

pragma once 方式：

pragma once是由编译器提供，保证一个文件不会被包含多次，这里的文件指的是物理文件，比如头文件(.h)

  • 优点：简单，直接加在每个头文件(.h)的最前面就好了

  • 缺点：不够灵活，比如两个头文件(a.h，b.h)，他们的内容是相同的。如果一个源文件(main.cpp)同时包含了这两个头文件(a.h，b.h)，那么就会出现重复包含的情况。

条件编译语句

条件编译通过定义宏，来避免头文件被多次编译

  • 缺点：比较麻烦，对于每一个条件编译语句都要定义个宏，而且条件编译语句写起来麻烦

  • 优点：灵活，条件编定义的作用范围，可以是一个文件的一部分，也可以是全部；也可以在其他的文件中也可以使用相同的宏。

头文件互相引用的问题：

问题出现原因：

头文件(a.h)中包含了 (b.h)，头文件(b.h)中包含了(a.h)

一般头文件互相包含的场景是：

头文件(a.h)中需要使用头文件(b.h)中的结构体或类(struct B)

头文件(b.h)中需要使用头文件(a.h)中的结构体或类(struct A)

前置声明

  • 声明和定义

    声明是不需要分配内存空间；定义是需要分配内存空间的。

    类或结构体的声明，就是声明了一个类或结构体的完成结构。

    类或结构体的定义，就是定义一个类或结构体的对象。

  • 类或结构体的前置声明(forward declaration)和声明

    (1)前置声明只是声明了一个类或结构体的名称，并不知道这个类或结构体的完整数据结构。

    一个前置声明的类或结构体，我们无法通过它来定义这个类或结构体的对象，因为不知道这个类或结构体的完整数据结构，所以无法计算出这个类或结构体对象所占用的内存空间。

    (2)声明不仅仅声明了一个类或结构体的名称，还声明了这个类或结构体的完整数据结构。

    一个声明的类或结构体，我们知道这个类或结构体的完整数据结构，可以用它来定义这个类或结构体的对象。

  • 不完全类型：

    前置声明的结构体或者类，在声明之前，称之为不完全类型，因为它没有完整的数据结构

  • 不完全类型的使用限制：

    (1)只能定义指向这种不完全类型的指针或引用

    因为不完全类型我们无法计算出空间大小，所以我们不能定义不完整类型的变量，但是因为指针的大小是确定的，所以我们可以定义不完全类型的指针或引用

    (2)只能声明以不完全类型作为参数或者返回类型的函数

    因为函数的声明是不需要分配空间的，所以我们可以使用不完成类型作为函数的参数或者返回类型；

    但是函数的定义，我们需要知道每个变量占用的空间大小，这样才能给这个函数分配空间，显然不完全类型的变量是无法计算空间大小的，所以我们不能定义以不完全类型作为参数或者返回类型的函数

解决方式

我们在b.h中对struct A进行前置声明，从而解决了头文件(.h)互相包含的问题

对于struct A的定义，我们可以通过在源文件(b.c)中，引用头文件(a.h)，来得到struct A的定义。
